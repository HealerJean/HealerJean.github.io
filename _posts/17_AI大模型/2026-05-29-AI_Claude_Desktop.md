# 一、功能菜单

![image-20260707134428347](/Users/zhangyujin1/Desktop/HealerJean/HCode/HealerJean.github.io/blogImages/image-20260707134428347.png)



## 1、左侧导航栏（全局项目管理）

### 1）核心标签页

- **`Cowork`**：协作模式，支持多人实时共享会话、共同编辑指令和上下文，适合团队协作场景。
- **`Code`**：代码模式，自动加载项目代码结构，支持代码补全、重构、调试和文件操作，

### 2）功能菜单

- **`New task`**：创建独立任务会话，每个任务可绑定不同上下文、指令和模型，适合拆分项目子需求。
- **`Projects`**：项目列表，切换不同项目（当前为 `baoxian-merchant-route`），每个项目独立保存上下文、记忆和配置。
- **`Artifacts`**：查看 / 管理项目生成的产物（如代码文件、文档、脚本），可直接打开、编辑或导出。
- **`Scheduled`**：定时任务面板，设置重复执行的 `AI` 任务（如周报生成、代码巡检、文档同步）。
- **`Customize`**：自定义项目配置，包括模型选择、指令模板、权限控制和界面布局。



### 3）`Recents`（最近会话）

- 快速访问历史对话，显示会话名称、时间和摘要，方便回溯之前的项目讨论。
- 底部 `Gateway`：全局模型 / 服务切换入口，可选择不同模型（如 Sonnet、Haiku）或接入自定义服务。



## 2、右侧配置面板（项目级设置）

### 1）`Instructions`（项目指令）

- 作用：定义 `Claude` 在当前项目中的**行为规则和工作流**，相当于给 AI 一份项目说明书。
- 特点：
  - 永久生效：所有后续对话都会遵循该指令。
  - 可编辑：点击铅笔图标修改规则，适合迭代优化工作流。
  - 示例用途：
    - 代码项目：要求先做代码评审、再写单元测试、最后生成文档。
    - 日程项目：像你当前这样，定义 AI 调整日程前的确认流程。

### 2）`Scheduled`（定时任务）

- 作用：设置**周期性自动任务**，让 Claude 按计划执行操作。
- 支持场景：
  - 每日 / 每周生成项目进度报告。
  - 定时巡检代码库，修复 lint 错误。
  - 同步知识库与代码变更。
- 操作：点击 `+` 创建新任务，设置执行频率、触发条件和执行指令。

### 3）`Context`（项目上下文）

- 作用：绑定项目相关的**文件、代码库、文档**，让 `Claude` 能直接读取和理解项目内容。
- 当前已绑定：`baoxian-merchant-route`（你的项目代码库），`Claude `可自动扫描文件结构、依赖关系和业务逻辑。
- 特点：
  - 只读访问：默认仅读取文件，不会直接修改代码（需你授权）。
  - 增量更新：项目文件变更后，`Claude` 会自动感知并更新上下文。



### 4）`Memory`（项目记忆）

- 作用：让 `Claude` 记住**项目关键信息和个人偏好**，避免重复询问。
- 示例记忆内容：
  - 业务规则：“商户路由系统的核心是保障高可用”。
  - 个人习惯：“我习惯每天下午 2-4 点为专注时间”。
  - 技术栈：“项目使用 React + TypeScript + Tailwind CSS”。
- 特点：记忆会跨会话持久保存，所有新对话都会自动引用这些信息。



## 3、`Cowork` vs `Code` 模式

| 模式       | 核心定位              | 设计目标                                                     |
| ---------- | --------------------- | ------------------------------------------------------------ |
| **Cowork** | 通用协作 / 任务管理   | 面向**全场景协作**：日程规划、文档撰写、项目管理、团队沟通等，代码只是其中一种能力 |
| **Code**   | 专业代码开发 / 工程化 | 面向**开发者**：深度集成代码库、文件系统、终端命令，专注于写代码、改代码、调试、重构 |

- **`Cowork` = 项目级协作助手**：管项目、管任务、管规则，代码只是其中一个小能力

- **`Code` = 专业代码开发助手**：管代码、管文件、管工程，和 OpenCode 一脉相承



### 1）功能与交互差异

#### a、`Cowork` 模式

- **上下文绑定**：绑定项目级「Instructions/Memory/Context」，适合长期项目的规则沉淀
- **能力侧重**：
  - 支持自然语言任务（如日程优化、文档生成）
  - 代码能力是**辅助**：可以生成代码片段，但不会直接操作本地文件
  - 侧重「协作 / 规划 / 记忆」，比如让 AI 记住你的工作习惯、项目规则
- **界面特点**：
  - 有 `Scheduled`（定时任务）、`Memory`（项目记忆）、`Instructions`（项目指令）等配置面板
  - 适合团队协作、长期项目管理、非编码类任务

#### b、`Code` 模式

- **上下文绑定**：直接绑定**本地代码目录**（如 `baoxian-merchant-route`），`Claude` 能实时读取 / 修改项目文件
- **能力侧重**：
  - 核心是**代码工程化**：自动扫描项目结构、依赖关系、代码语义
  - 支持**直接读写文件**、执行终端命令（如 `npm install`、`git` 操作）、运行测试
  - 可以理解完整代码库，做跨文件重构、Bug 修复、性能优化
- **界面特点**：
  - 无 `Scheduled`/`Memory` 等通用配置，专注于代码交互
  - 会显示文件树 / 代码预览（如果开启），支持直接在对话里修改代码
  - 和 `OpenCode` 的能力高度对齐，是「本地代码 + AI 自动操作」的专业模式

### 3）典型使用场景对比

| 场景                                     | 推荐模式   | 原因                                                     |
| ---------------------------------------- | ---------- | -------------------------------------------------------- |
| 让 AI 帮你优化日程、写周报、管理项目任务 | **Cowork** | 适合通用协作，能记住你的偏好和规则，长期迭代             |
| 开发新功能、修复 Bug、重构代码           | **Code**   | 能直接操作本地文件，理解完整代码库，专业代码开发效率更高 |
| 团队多人协作同一个项目                   | **Cowork** | 共享 Instructions 和 Context，统一团队协作规则           |
| 快速调试一个脚本、看某个库的源码实现     | **Code**   | 直接绑定目录，AI 能立刻读取并分析代码                    |



#### a、`Cowork` 协作模式（多人团队实战）

> 适用：需求评审、联合排障、团队代码方案讨论。

**场景**：后端小组 3 人一起梳理商户路由需求、定位线上 bug

1. 组长新建 Cowork 共享会话，把 `baoxian-merchant-route` 项目上下文绑定进来；
2. 组员全部加入，所有人实时看到提问、AI 输出、代码分析；
3. 一人贴报错堆栈，另一人补充业务逻辑，AI 同步读取所有人输入；
4. 共同定稿接口文档后，一键导出共享给测试、产品，不用转发聊天截图。



#### b、`Code` 代码模式（个人开发实战）

> 适用：本地项目编码、代码评审、脚本批量处理源码、bug 定位。

**场景**：批量提取 Filter 的 @Order 注解、重构过滤类、写单元测试

1. 切换 Code 标签，自动加载项目完整目录、Java 类、注解、依赖；
2. 直接发指令：扫描 filter 目录所有 java，提取执行顺序并排序；
3. AI 自动调用 shell 读取文件、分析注解、生成重构代码；
4. 内置文件预览，不用切终端 / IDE，改完直接生成可复制代码片段。





### 4）核心使用场景总结

| 场景       | 对应功能模块                             | 操作示例                                              |
| ---------- | ---------------------------------------- | ----------------------------------------------------- |
| 代码开发   | `Code` 标签 + `Context `+ `Instructions` | 绑定项目代码库，指令要求 “先做代码评审，再写单元测试” |
| 日程管理   | 对话区 +` Instructions`                  | 输入日程调整规则，让 AI 帮你优化时间分配              |
| 团队协作   | `Cowork` 标签 + `Instructions`           | 共享项目会话，统一团队协作规则                        |
| 自动化运维 | `Scheduled` + `Context`                  | 定时执行代码巡检、知识库同步任务                      |
| 长期项目   | `Memory `+ `Projects`                    | 记住项目关键信息，跨会话持续迭代                      |

# 二、三种模式

> `Claude Desktop` 是 `Anthropic` 官方出品的桌面客户端，支持 **macOS**、**Windows**、**Linux（Beta）**。

```
Claude Desktop
├── Chat      ← 日常对话、问答、写作、头脑风暴
├── Cowork    ← 复杂多步任务：研究、分析、整理文档（需 Pro+）
└── Code      ← 软件工程：编码、测试、调试、部署（需 Pro+，逐步推送中）
```

## 1、快速对比

|              | Chat               | Cowork               | Code                 |
| ------------ | ------------------ | -------------------- | -------------------- |
| **定位**     | 快速问答、迭代写作 | 复杂持续工作         | 软件开发             |
| **文件访问** | 手动上传/拖拽      | 授权文件夹读写       | 完整项目访问         |
| **运行方式** | 一次性对话         | 独立后台执行多步任务 | `Agent` 驱动编译运行 |
| **适用人群** | 所有人             | 知识工作者           | 开发者               |
| **子代理**   | ❌                  | ✅                    | ✅                    |
| **定时任务** | ❌                  | ✅                    | ❌                    |
| **沙箱**     | —                  | 隔离 VM              | 本地权限控制         |



## 2、`Chat` 模式

`Chat` 是 `Claude Desktop` 的基础模式，与 `claude.ai` 功能一致，额外多了桌面专属能力。

### 1）桌面专属特性

| 功能                          | 说明                             |
| ----------------------------- | -------------------------------- |
| **`Quick Entry`（快速入口）** | `macOS` 上双击 **`Option`** 键   |
|                               | 弹出紧凑窗口，悬浮在任何应用之上 |
|                               | 不离开当前工作即可提问           |
| **截图与窗口分享**（`macOS`） | 截取屏幕或分享窗口               |
|                               | Claude 直接看到你看到的内容      |
| **听写**（macOS）             | 用语音代替打字                   |
|                               | 适合边走边想或不想打字时         |
| 桌面连接器（Connectors）      | 连接本地工具和服务               |
|                               | 扩展 Claude 的能力边界           |

### 2）基础操作

- 输入文字提问
- 拖拽文件/图片到窗口
- 粘贴代码、文档、链接
- 多轮对话，上下文保持

## 3、`Cowork` 模式

> `Cowork` 是 Claude 的"准员工"模式——给出一个任务，Claude 自主完成多步工作。

### 1）能做什么

- **研究简报**：从多个来源搜集信息，撰写分析报告
- **文档整理**：对文件夹中的文件进行分类、重命名、汇总
- **电子表格**：创建带公式的 Excel，从杂乱数据生成结构化表格
- **演示文稿**：从分散资料生成 PPT
- **定时任务**：每日简报、每周汇总、邮件分类（需要 App 保持运行）

### 2）工作方式

1. 给 `Cowork` 一个文件夹路径
2. `Cowork` 阅读文件、分析问题
3. 在侧边栏展示执行计划
4. 自主执行多步操作（可并行）
5. 完成后通知你

### 3）`Claude in Chrome`（浏览器操控）

`Cowork` 模式下 `Claude` 可操控浏览器访问网页：

- `Claude `打开 `Chrome` 浏览器窗口
- 自主导航、填写表单、提取信息
- 适合网页数据采集、SaaS 操作自动化
- 需要你的监督，复杂交互可能需人工介入

### 4）安全机制

- **隔离虚拟机**中运行
- 只能读写你授权的文件夹
- 网络访问遵循你的出站设置



## 3、`Code` 模式

> `Code` 模式在桌面 `App` 中内置了 `Claude Code` 编程环境。

### 1）本地开发与云端开发

|                | 本地                                | 云端                                |
| -------------- | ----------------------------------- | ----------------------------------- |
| **工作方式**   | 选择本地文件夹，Claude 直接操作文件 | 连接 GitHub 仓库，Claude 在云端工作 |
| **开发服务器** | ✅ 可在浏览器预览                    | ❌                                   |
| **关 App 后**  | 任务中断                            | 任务继续                            |
| **适用**       | 日常开发、快速原型                  | 大型重构、长时间运行                |

### 2）三种交互模式

| 模式     | 文件修改 | 命令执行 | 适用场景           |
| -------- | -------- | -------- | ------------------ |
| **Ask**  | 逐次审批 | 逐次审批 | 新手、关键项目     |
| **Code** | 自动     | 逐次审批 | 日常开发           |
| **Plan** | ❌ 仅规划 | ❌        | 复杂任务先设计方案 |



### 3）三种视图模式

| 视图          | 详情程度     | 适用场景                   |
| ------------- | ------------ | -------------------------- |
| **`Verbose`** | 显示完整输出 | 调试、查错、需要完整上下文 |
| **`Normal`**  | 关键信息摘要 | 日常开发                   |
| **`Summary`** | 仅最终结论   | 快速验收、Review           |



### 4）界面特性

- 可视化 Diff 对比
- 内置终端（运行测试/构建）
- 内置文件编辑器（直接修改）
- 可拖拽布局（终端/预览/Diff/对话面板自由拖拽排列）
- `Git` 版本追踪，随时回退



### 5）`Side` `Chat`（侧聊）

- 不污染主线程，在侧边栏独立提问
- 适合临时查询、代码解释、文档查找
- 不影响主会话的状态和上下文



### 6）会话侧栏筛选

`Code` 模式左侧会话列表支持多维筛选：

| 筛选维度     | 选项                                     |
| ------------ | ---------------------------------------- |
| **状态**     | `Active`（进行中）/ `Archived`（已归档） |
| **环境**     | `Local`（本地）/ `Cloud`（云端）         |
| **项目分组** | 按项目目录自动归类                       |



### 6）`SSH` 远程

Code 模式支持 SSH 连接远程机器执行任务：

- 通过 SSH 连接到远程服务器
- 在远程机器上执行编译、部署等操作
- 与本地开发一致的体验
- 需可用的 SSH 凭据和网络连接

# 三、扩展与集成

## 1、`MCP` 扩展配置

> `MCP`（`Model Context Protocol`）是 `Claude Desktop` 最强大的特性，让 Claude 直接连接你的本地工具和数据源。

配置文件位置：**macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`

### 1）常用 MCP 服务器示例

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/yourname/projects", "/Users/yourname/documents"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "<your-token>"
      }
    },
    "brave-search": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-brave-search"],
      "env": {
        "BRAVE_API_KEY": "<your-key>"
      }
    },
    "sqlite": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-sqlite", "/Users/yourname/data.db"]
    }
  }
}
```



### 2）配置要点

1. 修改 `claude_desktop_config.json` 后 **必须重启 App** 才能生效
2. 建议从 **单个 MCP 服务器** 开始（如 filesystem），确认连通后再添加更多
3. 运行超过 5-8 个 MCP 服务器会明显增加启动时间和内存
4. 配置成功后，对话输入框附近会出现工具图标



### 3）推荐入手顺序

| 顺序 | MCP 服务器            | 用途                   |
| ---- | --------------------- | ---------------------- |
| 1    | `filesystem`          | 读写本地文件，浏览项目 |
| 2    | `github`              | 仓库、PR、Issue 操作   |
| 3    | `brave-search`        | 实时网页搜索           |
| 4    | `sqlite` / `postgres` | 数据库自然语言查询     |

> MCP 是开放标准，任何人都可以用 Python 或 TypeScript 编写自定义服务器，50 行代码内即可接入内部 API 或业务系统。



## 2、`Extensions`（扩展目录）

> 桌面版设置内置了扩展目录，可浏览和安装经 `Anthropic` 签名的扩展。

### 1）与 `MCP` 的关系

|              | MCP 服务器     | Extensions 扩展                |
| ------------ | -------------- | ------------------------------ |
| **来源**     | 自行配置或社区 | 经 Anthropic 签名的官方/第三方 |
| **安装方式** | 手写 JSON 配置 | 设置中搜索一键安装             |
| **安全性**   | 信任配置者     | 签名验证 + 加密存储            |
| **更新**     | 手动           | 自动                           |

### 2）安全机制

- **代码签名**：每个扩展包经过 Anthropic 签名，防止篡改
- **加密存储**：扩展的敏感数据（Token、密钥）以加密方式存储
- **权限声明**：安装时展示扩展需要的权限，经确认后才授予
- **运行隔离**：扩展在独立进程中运行，不会影响 App 主进程

## 3、`Skills`（桌面 `Skills`）

> `Claude Desktop` 的 `Skills` 不同于 `OpenCode` 的 `SKILL.md` 系统：

- 预定义的行为模式集合，指导 `Claude` 如何响应
- 可在设置中管理已安装的 `Skills`
- 第三方开发者可打包 `Skills` 并通过 `Extension`s 分发
- 与 `Projects` 的自定义指令互补：`Skills` 控制行为风格，自定义指令控制领域知识
- 当前主要为内置功能，第三方开发仍在完善中





# 四、其他特性

## 1、`Projects` 项目管理

> `Projects `是命名工作空间，跨会话保持上下文：

- **自定义指令**：设置一次，该项目的所有对话自动生效（如写作风格、编码规范）
- **文件附件**：架构文档、`API` 参考等跨会话可用
- **对话历史**：新会话可引用旧会话，真正做到"接着上次讨论"

适用场景

- 长期项目的技术文档与 `Claude` 共享
- 保持一致的风格和规范
- 团队知识库构建



### 1）项目会话：**会话持久化存储**

- 所有项目内的对话、指令、上下文引用都会被**本地缓存 + 云端同步**保存：
  - 本地：Mac 路径 `~/Library/Caches/Claude/`，Windows 路径 `%LOCALAPPDATA%\Claude\`，以 `.jsonl` 格式存储会话历史
  - 云端：绑定你的 Anthropic 账号，跨设备登录后可同步加载
- 每个项目独立隔离：切换项目时，只会加载该项目的会话历史，不会互相干扰
- 会话可回溯：在左侧 `Recents` 里能看到所有历史对话，点击即可恢复当时的上下文状态



### 2）**项目会话：上下文截断与智能管理**

> `Claude` 不会把**完整会话历史**都塞进每一次请求，而是通过以下方式控制 Token 消耗：

- **滑动窗口机制**：只保留最近 N 轮对话作为上下文，更早的历史会被压缩或移出上下文窗口
- **摘要化处理**：对旧对话自动生成摘要，用短摘要替代长文本，大幅减少 Token 占用
- **选择性加载**：只加载和当前查询相关的历史片段，无关内容会被暂时忽略
- **项目级记忆**：把关键信息（如业务规则、技术栈）存在 `Memory` 里，用固定长度的记忆条目替代大量重复对话，进一步压缩 Token



## 2、`Computer Use`（计算机使用）

> Beta 功能，需在设置中手动开启。

`Claude` 可以：

1. 截取屏幕截图
2. 分析屏幕内容
3. 控制鼠标和键盘操作
4. 验证操作结果

**现实场景：**

- 填写表单（从文档中提取数据填入网页）
- 按规范重命名和整理文件
- 在多步骤业务流程中操作桌面应用

**注意：**

- 比手动操作慢
- 复杂 UI 可能出错
- 需要你监督和纠正

> 建议在受控环境中使用，避免同时打开密码管理器、银行应用等敏感界面。
