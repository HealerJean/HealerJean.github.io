# 一、基本介绍

`OpenCode` 桌面版是终端 TUI 的图形化版本，提供了同样的 AI 编程代理能力，但拥有更友好的图形界面。

## 1、界面导览

桌面版主界面分为以下几个区域：

```
┌─────────────────────────────────────────────────┐
│  菜单栏                  会话标签  [+] 设置 │
├──────────┬──────────────────────────────────────┤
│          │                                      │
│  文件树   │        对话/编辑主区域                │
│  (侧边栏) │                                      │
│          │    ┌──────────────────────────┐       │
│  src/     │    │  代码编辑区               │       │
│  ├─ index │    │  (语法高亮、行号)          │       │
│  ├─ utils │    └──────────────────────────┘       │
│  tests/   │                                      │
│          │    ┌──────────────────────────┐       │
│          │    │  AI 对话区                │       │
│          │    │  输入消息...              │       │
│          │    └──────────────────────────┘       │
├──────────┴──────────────────────────────────────┤
│  状态栏: Provider | Model | 会话信息             │
└─────────────────────────────────────────────────┘
```

**主要区域说明：**

| 区域           | 说明                                     |
| -------------- | ---------------------------------------- |
| **文件树**     | 左侧边栏，显示项目文件结构，点击打开文件 |
| **代码编辑区** | 居中/右侧，语法高亮、行号，支持编辑      |
| **AI 对话区**  | 底部/右侧，输入 Prompt，查看 AI 回复     |
| **Diff 视图**  | AI 修改代码时显示前后对比                |
| **会话标签**   | 顶部 Tab，管理多个并行会话               |



## 2、桌面版 vs `CLI` 对比

| 特性           | 桌面版       | CLI (终端)         |
| -------------- | ------------ | ------------------ |
| **图形界面**   | ✅ 原生窗口   | ❌ 纯终端           |
| **内置编辑器** | ✅ 语法高亮   | ❌ 外部编辑器       |
| **Diff 视图**  | ✅ 图形化对比 | ✅ 终端 Diff        |
| **文件树**     | ✅ GUI 导航   | ❌ 命令导航         |
| **多会话**     | ✅ Tab 标签页 | ✅ 子会话           |
| **拖拽图片**   | ✅ 原生支持   | ✅ 终端支持         |
| **会话分享**   | ✅            | ✅                  |
| **远程访问**   | ❌            | ✅ `opencode serve` |
| **CI/脚本**    | ❌ 有限       | ✅ 原生支持         |
| **资源占用**   | 较高         | 较低               |
| **系统托盘**   | ✅            | ❌                  |
| **自动更新**   | ✅ GUI 提示   | ✅ 命令行提示       |
| **快捷键**     | 桌面标准键位 | `Ctrl+X` 系列      |

**选择建议：**

- 日常开发、代码审查、学习 — 用 **桌面版**
- CI 集成、远程协作、脚本自动化 — 用 **CLI**
- 两者可同时使用，配置共享



## 3、键盘快捷键

| 快捷键         | 作用                 |
| -------------- | -------------------- |
| `Cmd/Ctrl + N` | 新会话               |
| `Cmd/Ctrl + O` | 打开项目文件夹       |
| `Cmd/Ctrl + S` | 保存文件             |
| `Cmd/Ctrl + F` | 文件内搜索           |
| `Cmd/Ctrl + B` | 切换侧边栏           |
| `Cmd/Ctrl + ,` | 打开设置             |
| `Cmd/Ctrl + Q` | 退出                 |
| `Tab`          | 切换 Build/Plan 模式 |
| `↑` / `↓`      | 浏览历史消息         |
| `Esc`          | 取消当前操作         |



# 二、配置

## 1、配置 `Provider`

### 1）首次启动配置

打开桌面版后，第一步是连接 AI 模型 Provider：

1. 点击 **设置** (Cmd/Ctrl + `,`)
2. 进入 **Provider** 选项卡
3. 选择 Provider 并输入 API Key

### 3）支持的方式

| 方式                 | 说明                               |
| -------------------- | ---------------------------------- |
| **OpenCode Zen**     | 内置免费模型，推荐新手             |
| **GitHub Copilot**   | 直接用 Copilot 订阅 OAuth 登录     |
| **ChatGPT Plus/Pro** | 直接用 OpenAI 订阅 OAuth 登录      |
| **Anthropic API**    | 手动输入 Claude API Key            |
| **OpenAI API**       | 手动输入 GPT API Key               |
| **Ollama**           | 连接本地模型                       |
| **自定义**           | 任意 OpenAI 兼容接口（含代理中转） |

> **你的配置参考：** 如果使用代理中转，在 Provider 设置中选择 `OpenAI`，将 `baseURL` 设为你的代理地址（如 `http://127.0.0.1:8787/opencode`），模型名填入实际模型 ID。

### 2）切换模型

在状态栏点击当前模型名，或通过设置中的 **Model** 下拉框选择。



## 2、初始化项目

### 1）打开项目

1. 点击 **File → Open Folder** (Cmd/Ctrl + O)
2. 选择项目目录
3. 文件树自动加载项目文件

### 2）运行 /init

在 AI 对话区输入：

```
/init
```

这会分析项目结构，创建 `AGENTS.md` 文件，让 AI 了解你的项目编码规范、构建命令等。

> 建议将 `AGENTS.md` 提交到 Git 仓库，团队共享。

## 3、核心操作

### 1）基本工作流

```
1. Cmd+O 打开项目
2. /init 初始化（首次）
3. Tab 切换 Build/Plan 模式
4. 输入任务描述
5. 审查 AI 的 Diff 修改
6. 确认/调整/保存
```



### 2）`Plan` 模式 vs `Build` 模式

桌面版右下角状态栏显示当前模式，点击或按 **Tab** 切换：

|          | Plan 模式                    | Build 模式           |
| -------- | ---------------------------- | -------------------- |
| 图标     | 📋                            | 🔨                    |
| 读取文件 | ✅                            | ✅                    |
| 修改代码 | ❌（仅建议）                  | ✅                    |
| 执行命令 | ❌                            | ✅                    |
| 适用     | 分析代码、设计方案、代码审查 | 编码实现、重构、调试 |



### 3）引用文件

在对话中输入 `@` 触发模糊搜索，选择要引用的文件：

```
这个函数在 @utils/helpers.ts 里是怎么实现的？
```

AI 会自动读取文件内容作为上下文。



### 4）拖拽图片

直接拖拽图片到桌面版窗口，AI 能看到设计稿、架构图或错误截图。



### 5）审查 Diff

AI 修改代码后，桌面版会显示 Diff 对比视图：

- **绿色** = 新增代码
- **红色** = 删除代码
- 可以逐行审查，确认后再保存

### 6）撤销修改

```
/undo
```

支持多次撤销，恢复文件到修改前的状态。

# 三、桌面版特色功能

## 1、多会话并行

顶部的会话标签页支持多个并行会话：

```
[会话1: 订单模块*] [会话2: 用户认证] [+] 
```

- 不同任务用不同会话，上下文不混淆
- 可以同时进行前端重构和后端调试

## 2、文件树与快速跳转

- 左侧文件树点击即可打开文件
- Cmd/Ctrl + P 快速搜索文件名
- 支持多标签页打开文件



## 3、内置代码编辑器

- 语法高亮（支持主流语言）
- 行号显示
- 直接在桌面版内编辑代码
- AI 修改实时 Diff 展示
